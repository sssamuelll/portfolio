# content-raw — index

Raw material for sdar.dev. Heavy files (photos, videos, audio) are gitignored — only this index, the folder skeleton, and small handwriting scans get committed. The build pipeline (TBD, phase 3.5) will read processed copies from `public/media/` or similar; this folder stays as the working set.

## How to fill this in

One row per piece. Add as you drop files. The agent uses this to know what's available without scanning binaries.

Conventions:
- **date**: `YYYY-MM-DD` (or `YYYY-MM` if exact day unknown). Sortable.
- **location**: short label — `Bali`, `Kagawa`, `Mannō`, `Nürnberg`, `New York`, `Madrid`, `Caracas`. One word when possible.
- **type**: `photo` · `video` · `audio` · `moment` · `handwriting`.
- **filename**: relative path under `content-raw/`.
- **caption**: a phrase, not a sentence. Lowercase. The voice of the site.

Empty caption is fine. Better blank than filler.
